# 76 numera/groebner_at_scale.iii — Implementation Spec

## Verdict
STUB — the gospel candidate body is a non-functional façade built on **fiction**. Its sole worker extern, `gb_v1_compute(polynomials:*u8, polys_len:u32, order:u8, out_basis:*u8, out_basis_cap:u32, out_len:*u32) from "groebner.iii"`, **does not exist** in the realized Module 18 (`groebner.iii`, see `DOCS/CONVERGENCE-SPECS/groebner.spec.md`): Module 18's surface is entirely **slot-id based** (`gb_init(nvars)`, `gb_begin(arena,p)`, `gb_new_poly`, `gb_append_term(slot,coeff_bid,exps)`, `gb_buchberger(generators:*u32,gen_n,out_basis:*u32,out_n:*u32)`, `gb_reduce`, `gb_lead_coeff`, `gb_basis_digest`) — it has **no `*u8` polynomial-system codec and no `gb_v1_compute`**. Consequently the gospel `gb_basis` cannot link, and `gb_verify` is an outright no-op (`return GB_OK` with zero checking) — directly violating the module's own headline contract ("the produced basis is verified by independent checking of every pair reduction... the verifier transcript enters the chain") and M12/M17. The maximal realization makes Module 76 the genuine **large-system verified wrapper**: it (1) deserializes a deterministic flat `*u8` polynomial-system format into Module 18 slots, (2) enforces a declared cost budget (M19), (3) drives Module 18's bounded Buchberger (which already yields the canonical *reduced* basis), (4) **independently verifies** the result by re-deriving ideal membership of every original generator modulo the output basis (a check Buchberger never performs on the originals) plus digest reproducibility, (5) serializes the basis back to `*u8`, and (6) emits the verifier transcript into the witness chain via `wh_publish` + `ws_register`. Six mandate/law defects and four fictional/absent dependencies are closed below.

## Purpose
`numera_groebner_at_scale` IS the industrial-scale, **independently-verified** front end to Gröbner-basis synthesis over `GF(p)[x_0..x_{nvars-1}]`. It owns the serialized boundary (a flat little-endian polynomial-system blob in, a flat reduced-basis blob out), enforces a caller-declared cost vector, delegates the actual closure to Module 18's bounded Buchberger (NIH — no field math or S-pair logic is re-implemented here, M1), and then *re-checks the answer from scratch* — every input generator must reduce to zero modulo the returned basis (ideal-membership), and the basis digest must recompute byte-identically. Only a verified basis is serialized and chained; the verdict + per-generator transcript enters the witness spine, giving the synthesized ideal a checkable certificate (M12) recomputable byte-for-byte from recorded inputs (M10). **Hexad: kind_cognition. Ring: R0. K_synth: 0.95** (allocation in the field/bigint/slot layer may fail; verification may refuse).

## Public API
All public fns return a status: `i32` negative-error codes (W9/W12), compared only by `==`/`!=` (W11/Trap 3). The gospel's 7-/5-param byte-blob signatures are **replaced** by a W2-clean session + aggregate-pointer surface (mandate-driven; the only API divergence — justified in Gap §2). Output and "request" pointers are single `*u8` aggregates whose internal layout folds the gospel's `out_basis`/`out_cap`/`out_len` (and the `arena`/`p`/`order`) down to ≤4 params per fn.

```
fn gbs_init() -> i32 @export
fn gbs_session(arena: u64, p: u64, order: u8) -> i32 @export
fn gbs_solve(system: *u8, sys_len: u32, budget: *u8, out: *u8) -> i32 @export
fn gbs_verify(system: *u8, sys_len: u32, basis: *u8, basis_len: u32) -> i32 @export
fn gbs_digest(basis: *u8, basis_len: u32, out32: *u8) -> i32 @export
fn gbs_witness_last(out_frag_id: *u8) -> i32 @export
```

Return-status convention per fn:
- `gbs_init` / `gbs_session` → `i32` ∈ {`GBS_OK`(0), negatives}. `gbs_session` validates `order` (only graded-lex `0u8` accepted, M4 — no silent alternate order) and stores the field session; the actual `gb_init(nvars)` is issued inside `gbs_solve` once `nvars` is read from the input header.
- `gbs_solve` → `i32` ∈ {`GBS_OK`(0), `GBS_E_NULL`, `GBS_E_NOSESS`, `GBS_E_FORMAT`, `GBS_E_BUDGET`, `GBS_E_FULL`, `GBS_E_VERIFY`, `GBS_E_WITNESS`}. On `GBS_OK` the verified reduced basis has been serialized into `out` and its transcript chained; on any negative the output aggregate's `len` field is set to `0u32` and nothing is chained (atomic: refuse, do not partially emit — M5).
- `gbs_verify` → `i32`: `GBS_OK` iff the supplied `basis` is an independently-verified reduced Gröbner basis of the ideal generated by `system` (every generator reduces to 0; basis monic + inter-reduced); `GBS_E_VERIFY` if any check fails; `GBS_E_FORMAT`/`GBS_E_NULL`/`GBS_E_NOSESS`/`GBS_E_FULL` otherwise. This is the **real** content of the gospel's no-op `gb_verify`.
- `gbs_digest` → `i32`: writes the 32-byte content address of the serialized basis into `out32`; `GBS_E_FORMAT`/`GBS_E_NULL` on malformed input. Equals the value Module 18's `gb_basis_digest` would produce for the same ideal (re-derived over the serialized form so an external party needs only the bytes).
- `gbs_witness_last` → `i32`: copies the 32-byte fragment id emitted by the most recent successful `gbs_solve` into `out_frag_id`; `GBS_E_BAD` if no fragment has been emitted since `gbs_init`. (W12 — exposes the chain anchor.)

## Algorithm
Common conventions: every public op (except `gbs_init`) first checks `GBS_INITED == 1u8` (else `GBS_E_NOSESS` for solve/verify/digest paths that need a session — `gbs_init` itself is the bootstrap). `gbs_solve`/`gbs_verify`/`gbs_digest` additionally require `GBS_SESS == 1u8`. No `%` anywhere (Trap 11 not exposed). All loops use the **W14 sentinel/flag** form — no `break`, no `&&` (compound `while` is ungrammatical; flags drive a single `while flag == Nu8`). No recursion (W15): the only naturally recursive work (Buchberger closure, multivariate reduction) lives *inside Module 18*; Module 76's loops are flat bounded scans. `u32`/`u64` ordering compares are legal (Trap 3 is signed-only); the module returns/reads no `i64` and compares no `i32` with `<`/`>`.

**Serialized blob format (deterministic, little-endian, byte-addressed — M2/W5).**
System blob: `u32 GBS_MAGIC` ‖ `u32 nvars` ‖ `u32 n_polys` ‖ for each poly: `u32 n_terms` ‖ for each term: `u64 coeff_value` ‖ `nvars × u32 exponent`. Basis blob: identical shape (the magic/nvars/n_polys header + per-poly term lists). All multi-byte words are read/written **byte-by-byte through `*u8`** with explicit shift/mask (`(w >> (i*8)) & 0xFFu64`) so no `*u32`/`*u64` store-width bug can occur (Trap 5) and so layout is endianness-pinned regardless of host. A header `nvars > GBS_MAX_VARS`, `n_polys > GBS_MAX_GENS`, wrong magic, or a length that does not exactly cover the declared terms ⇒ `GBS_E_FORMAT` (no over-read).

**Budget format (M19 cost lattice).** `budget` is `GBS_BUDGET_WORDS` (=4) little-endian `u64`: `max_input_terms`, `max_basis_out`, `max_total_terms`, `reserved`. Bounds are on **structural counts only** (never wall-clock) so the budget gate is itself deterministic (M2). Enforcement points: (a) the running total of input terms parsed must stay `<= max_input_terms` else `GBS_E_BUDGET`; (b) `*out_n` after Buchberger must be `<= max_basis_out` else `GBS_E_BUDGET`; (c) Module 18's own `GROEBNER_E_FULL` (its static caps) is surfaced as `GBS_E_FULL` — the hard ceiling that converts any over-cap pathological input into an honest refusal rather than a wrong answer (M4/M5/M19). All three comparisons are `u64` ordering (legal).

**`gbs_init()`** — idempotent: if `GBS_INITED == 1u8` return `GBS_OK`. Else fill `GBS_PRODUCER`/`GBS_OPID` with the fixed 32-byte module identifiers (byte stores of the ASCII tag, zero-padded), clear `GBS_HAVE_FRAG`, set `GBS_INITED=1u8`. Pure array writes ⇒ deterministic; bounded (64 byte writes) ⇒ M19.

**`gbs_session(arena, p, order)`** — require `GBS_INITED==1u8` else `GBS_E_NOSESS`; require `order == GBS_ORDER_GRLEX` else `GBS_E_FORMAT` (M4 — refuse any unsupported order, never silently substitute). Store `GBS_ARENA=arena` (the allocation capability, M8), `GBS_MOD_P=p`, `GBS_ORDER=order`, set `GBS_SESS=1u8`. Reversible: a fresh `gbs_session` overwrites it; `gbs_init` does not clear it (deliberate — re-init is idempotent, session is replaceable). M8 note: `arena`+`p` are the field-session capability forwarded into Module 18 via `gb_begin`.

**`gbs_deserialize(system, sys_len, max_input_terms) -> i32` (internal).** Validate header (magic, `nvars<=GBS_MAX_VARS`, `n_polys<=GBS_MAX_GENS`). Store `GBS_NVARS=nvars`. Issue `gb_init(nvars)` then `gb_begin(GBS_ARENA, GBS_MOD_P)` (resets Module 18 to a clean bump epoch for this solve — the documented M18 lifecycle). Walk polys with a `pi` counter and a `bad` flag (W14): for each poly, `s = gb_new_poly()`; if `s == 0xFFFFFFFFu32` set `bad`/return `GBS_E_FULL`; record `GBS_GEN_SLOTS[pi]=s`. Walk its terms with a `ti` counter: read `coeff_value` (u64, byte-assembled), running `term_total += 1`; if `term_total > max_input_terms` ⇒ `GBS_E_BUDGET`; read `nvars` exponents into `GBS_TERM_EXPS`; `cb = bigint_from_u64(GBS_ARENA, coeff_value)` (if `cb==0u64` and `coeff_value!=0u64` ⇒ `GBS_E_FULL`, alloc failed); `rc = gb_append_term(s, cb, &GBS_TERM_EXPS[0])`; if `rc != GBS_OK`-equivalent (`gb` returns `GROEBNER_E_FULL`) ⇒ `GBS_E_FULL`. After all terms, `gb_normalize(s)`. Bounds offset reads against `sys_len` before every word (no over-read). Set `GBS_GEN_N=n_polys`. **Determinism:** fixed left-to-right parse, fixed slot allocation order; the bigint id values are deterministic given the arena (bigint.iii is deterministic). **Bounded** by `n_polys ≤ GBS_MAX_GENS` and `term_total ≤ max_input_terms` (M19).

**`gbs_serialize_basis(out) -> i32` (internal).** `out` is the output aggregate `u32 cap` ‖ `u32 len` ‖ payload-area. Compute required bytes from `GBS_OUT_SLOTS[0..GBS_OUT_N]` via `gb_poly_term_count`; if header+body `> cap` ⇒ `GBS_E_FORMAT` (caller under-provisioned), set `len=0`. Else write magic ‖ `GBS_NVARS` ‖ `GBS_OUT_N`, then per basis slot: `nt = gb_poly_term_count(slot)` ‖ for `k` in `0..nt`: `gb_term_at(slot, k, &coeff, &GBS_TERM_EXPS[0])` then byte-write `coeff` (u64 LE) ‖ `nvars` exponents (u32 LE each) — all through `*u8` (Trap 5). Write `len` last. Module 18 keeps each polynomial in descending graded-lex, and `gb_term_at` is index-ordered, so the serialized byte stream is canonical (M2/W5).

**`gbs_input_commit(system, sys_len)` (internal).** Fold the *entire input blob* with FNV-1a-64 (`GBS_FNV_OFF`, `GBS_FNV_PRIME`) into 8 bytes, then expand deterministically to 32 bytes by four independently-seeded FNV passes (seeds `0,1,2,3` mixed in), byte-stored into `GBS_IN_COMMIT`. This is the witness `in_commit` — a content address of the problem instance (M10: any party recomputes it from the same bytes). FNV is the hand-rolled NIH digest used elsewhere in the substrate (M1); it is not a security primitive here, only a reproducible commitment, so collision-resistance is not relied upon for soundness (soundness comes from the membership re-derivation below).

**`gbs_independent_verify() -> i32` (internal) — THE differentiator (M12/M17).** After Buchberger has written `GBS_OUT_SLOTS[0..GBS_OUT_N]`, re-derive correctness *without trusting the closure path*:
- **Ideal membership of every original generator.** Loop `gi` over `GBS_GEN_SLOTS[0..GBS_GEN_N]` with an `ok` flag (W14, `while ok == 1u8 && gi < n` rendered as nested flag-guard, no `&&`): `r = gb_reduce(GBS_GEN_SLOTS[gi], &GBS_OUT_SLOTS[0], GBS_OUT_N)`; if `gb_lead_coeff(r) != 0u64` set `ok=0u8` (a generator that does **not** reduce to 0 proves the returned set is *not* a Gröbner basis of the ideal — refuse); `gb_drop_poly(r)`; record the per-generator pass flag into `GBS_XSCRIPT`. Because every original generator is in the ideal by definition, a correct reduced basis must reduce each to 0; Buchberger never performs this reduction on the *originals*, so it is a genuinely independent check (M17 — never blindly trust the synthesis).
- **Monic + self-reduced spot-check.** For each basis slot, require `gb_lead_coeff(slot)` low-limb `== 1u64` (monic, via `gb_coeff_lo(slot,0u32)`) — the reduced-basis invariant Module 18 claims. (Full pairwise non-divisibility is *guaranteed* by M18's autoreduce and re-checking it would duplicate M18; the monic test is the cheap independent witness that autoreduce ran. The membership test above is the load-bearing soundness check.)
- **Digest reproducibility.** `gb_basis_digest(&GBS_OUT_SLOTS[0], GBS_OUT_N, &GBS_OUT_COMMIT[0])`; this both fills the witness `out_commit` and serves as the M10 reproducible certificate. If any check fails ⇒ `GBS_E_VERIFY`. **Determinism:** all reductions use Module 18's fixed first-divisor rule; the verdict is a pure function of (system, basis) (M2). **No recursion** — flat flag-driven scans; the recursive reduction is inside `gb_reduce`. **Bounded:** `GBS_GEN_N ≤ GBS_MAX_GENS` reductions, each itself M19-bounded by M18 (M19).

**`gbs_emit_witness(payload_len) -> i32` (internal).** Build the verdict transcript in `GBS_XSCRIPT`: `u32 n_generators` ‖ `u32 basis_n` ‖ `u8 verdict(1=verified)` ‖ `GBS_GEN_N × u8` per-generator reduced-to-zero flags (byte-stored, Trap 5). Then `fid_h = wh_publish(&GBS_PRODUCER[0], &GBS_OPID[0], &GBS_IN_COMMIT[0], &GBS_OUT_COMMIT[0], GBS_REVTAG, GBS_PHASE, GBS_PILLAR, &GBS_PRODUCER[0] /*0 antecedents*/, 0u32, &GBS_XSCRIPT[0], payload_len, &GBS_LAST_FRAG[0])`. If `fid_h == 0xFFFFFFFFFFFFFFFFu64` ⇒ `GBS_E_WITNESS` (chain full / not inited — refuse, emit nothing further, M5). Else `rc = ws_register(fid_h)`; if `rc != 0i32` ⇒ `GBS_E_WITNESS`. Set `GBS_HAVE_FRAG=1u8`. The fragment carries the reduced-basis content address as `out_commit` and the verifier transcript as payload — this is precisely the gospel's "verifier transcript enters the chain" (M6 witness continuity; the antecedent-free fragment is the root commitment for this synthesis op).

**`gbs_solve(system, sys_len, budget, out)`** — the orchestration (W2: 4 params, output folded into `out`):
1. Guards: `GBS_INITED`/`GBS_SESS`; null-check `system`,`budget`,`out` ⇒ `GBS_E_NULL`.
2. Read budget words (byte-assembled u64×4). `gbs_input_commit(system, sys_len)`.
3. `rc = gbs_deserialize(system, sys_len, max_input_terms)`; on negative, set `out.len=0`, return `rc` (covers `GBS_E_FORMAT`/`GBS_E_BUDGET`/`GBS_E_FULL`).
4. `rc = gb_buchberger(&GBS_GEN_SLOTS[0], GBS_GEN_N, &GBS_OUT_SLOTS[0], &GBS_OUT_N)`; if `rc != 0i32` map `GROEBNER_E_FULL→GBS_E_FULL`, else `GBS_E_VERIFY` for any other M18 error, set `out.len=0`, return.
5. Budget gate (b): `GBS_OUT_N > max_basis_out` ⇒ `out.len=0`, `GBS_E_BUDGET`.
6. `rc = gbs_independent_verify()`; if `rc != GBS_OK` ⇒ `out.len=0`, return `GBS_E_VERIFY` (refuse to serialize/chain an unverified basis — M5/M12).
7. `rc = gbs_serialize_basis(out)`; on `GBS_E_FORMAT` (cap too small) ⇒ return it (verified but not delivered; nothing chained — the caller retries with a bigger buffer; deterministic, no partial chain).
8. `rc = gbs_emit_witness(payload_len)`; on `GBS_E_WITNESS` ⇒ `out.len=0`, return (atomic: a delivered basis must be chained; if chaining fails we refuse the whole op). 
9. Return `GBS_OK`. **Atomicity (M5):** on every negative path nothing is chained and `out.len=0`; `GBS_OK` ⟺ (verified ∧ serialized ∧ chained).

**`gbs_verify(system, sys_len, basis, basis_len)`** — the real `gb_verify`. Guards as above. `gbs_input_commit` (for symmetry/logging is optional — not chained here). Deserialize `system` into `GBS_GEN_SLOTS` (`gbs_deserialize` with `max_input_terms = GBS_MAX_TERMS-equivalent`, i.e. no extra budget — bounded by M18 caps). Deserialize `basis` into `GBS_DG_BASIS[0..bn]` via the same per-poly reader (each poly → a fresh slot, normalized). Run the **membership + monic + digest** checks of `gbs_independent_verify` but against `GBS_DG_BASIS` instead of `GBS_OUT_SLOTS` (the basis came from the caller, not from our own Buchberger — fully independent, M17). Return `GBS_OK` iff every original generator reduces to 0 modulo the supplied basis and the basis is monic; else `GBS_E_VERIFY`. **This converts the gospel's `return GB_OK` no-op into an actual independent ideal-membership proof.**

**`gbs_digest(basis, basis_len, out32)`** — guards; deserialize `basis` into `GBS_DG_BASIS[0..bn]`; `gb_basis_digest(&GBS_DG_BASIS[0], bn, out32)`. Drop the rebuilt slots LIFO. Gives a stand-alone content address computable from bytes alone (M10).

**`gbs_witness_last(out_frag_id)`** — if `GBS_HAVE_FRAG == 0u8` ⇒ `GBS_E_BAD`; else copy 32 bytes `GBS_LAST_FRAG → out_frag_id`, return `GBS_OK`.

### Reentrancy / scratch non-aliasing (Trap 7 hoist safety)
Single-threaded; the only risk is a callee clobbering a caller's scratch. `gbs_solve → {gbs_deserialize, gb_buchberger, gbs_independent_verify, gbs_serialize_basis, gbs_emit_witness}`. Serialize and deserialize both use `GBS_TERM_EXPS`, but never concurrently (deserialize fully precedes Buchberger; serialize fully follows verify). The verifier uses a **distinct** `GBS_VER_EXPS` (declared but reads come from `gb_term_at`/`gb_reduce`, which write into M18's own scratch, not ours) so even if a future verify path serializes mid-loop the buffers are disjoint. `GBS_DG_BASIS` (verify/digest) and `GBS_OUT_SLOTS` (solve) are disjoint slot lists. M18's internal scratch (`GROEBNER_*`) is touched by no Module-76 array. Therefore no live Module-76 scratch is overwritten by any callee (W8: every buffer statically sized, bounds in §Data Structures).

## KAT Vectors (>= 3)
Field GF(7) (`p` = bigint id for 7), `order = GBS_ORDER_GRLEX (0)`, `nvars` per case. Blobs are the exact little-endian byte sequences. The self-test builds the input blob, runs the op, and checks the returned status, the output blob bytes, and (where chained) `gbs_witness_last` + `gbs_digest` byte-for-byte.

1. **Round-trip + already-reduced linear ideal (nvars=2, GF(7)).**
   System: 2 polys. `g1 = 1·x + 6` → terms `[(coeff 1, exp [1,0]), (coeff 6, exp [0,0])]`; `g2 = 1·y + 5` → `[(1,[0,1]),(5,[0,0])]`. Blob = `MAGIC, nvars=2, n_polys=2, [n_terms=2, (1,[1,0]),(6,[0,0])], [n_terms=2,(1,[0,1]),(5,[0,0])]`. Budget = `{max_input_terms=64, max_basis_out=64, max_total_terms=4096, 0}`.
   `gb_buchberger`: LTs `x`,`y` coprime ⇒ Criterion-1 skip ⇒ reduced basis `{x+6, y+5}`, `GBS_OUT_N=2`. **Verify:** `g1`,`g2` each reduce to 0 mod the basis (they ARE the basis) ⇒ verdict 1, both per-gen flags 1. Expected `gbs_solve → GBS_OK`; `out` blob = same shape with `n_polys=2` and the two polynomials byte-identical to input (monic, already reduced). Verifies deserialize→Buchberger→membership-verify→serialize round trip + the budget gate + a non-trivial witness emission (transcript bytes `n_gen=2, basis_n=2, verdict=1, 0x01, 0x01`).

2. **Buchberger produces a new element; full verified closure (nvars=2, GF(7)).**
   System: `f1 = x^2 + 1` → `[(1,[2,0]),(1,[0,0])]`; `f2 = x·y + 2` → `[(1,[1,1]),(2,[0,0])]` (Module-18 KAT 4 instance). Pair (f1,f2) shares `x` ⇒ not coprime; `S = 5x + y` reduces to itself, monic `x + 3y`, added; subsequent pairs reduce to 0; autoreduce yields the canonical reduced basis (the M18 spec freezes its exact tail + digest). **Verify:** `f1` and `f2` both reduce to 0 modulo the returned basis (they generate the ideal) ⇒ verdict 1. Expected `gbs_solve → GBS_OK`; `out` blob = the frozen reduced-basis bytes from the M18 reference run; `gbs_digest(out)` == `GBS_OUT_COMMIT` == the M18-frozen 32-byte hash (run twice → identical ⇒ M2/M10). Verifies the closure path, the independent membership re-derivation on a non-trivial basis, and digest reproducibility.

3. **Independent-verify REJECTS a wrong basis (nvars=2, GF(7)) — proves the negative.**
   `gbs_verify(system, basis)` with `system = {f1=x^2+1, f2=x·y+2}` (KAT 2) but a deliberately **wrong** `basis = {x + 3y}` only (omit the second canonical element). `f2 = xy+2` reduces modulo `{x+3y}` to a nonzero remainder (xy → −3y·y = 4y^2, plus 2; no further reduction by `x+3y`) ⇒ `gb_lead_coeff(remainder) != 0u64` ⇒ `ok=0`. Expected `gbs_verify → GBS_E_VERIFY` (-6). **This is the load-bearing test that the verifier actually verifies** (the gospel's no-op would wrongly return OK) — it proves the gate FAILS on a bad basis, per the "prove the negative case" discipline.

4. **Format + budget refusals (nvars=1, GF(7)).**
   (a) Truncated blob (declares `n_terms=3` but bytes cover only 2) ⇒ `gbs_solve → GBS_E_FORMAT` (-3), `out.len=0`, nothing chained. (b) `order=1u8` passed to `gbs_session` ⇒ `GBS_E_FORMAT`. (c) `budget.max_input_terms=1` with a 3-term system ⇒ `GBS_E_BUDGET` (-4) during deserialize, `out.len=0`. (d) `nvars=17` in the header ⇒ `GBS_E_FORMAT` (exceeds `GBS_MAX_VARS`). Verifies every refusal path returns the right negative code and chains nothing (M5 atomicity, no over-read).

(Phase 2: vectors 1 and 3–4 are hand-checkable; vector 2's frozen output bytes + 32-byte digest are inherited from Module 18's frozen reference execution and locked, per the crypto-KAT discipline used across numera.)

## Gap / Fix List (gospel is a STUB; every defect + fix)
1. **Fiction extern `gb_v1_compute from "groebner.iii"` (does not exist).** The realized Module 18 has no `gb_v1_compute` and no `*u8` codec — it is slot-id based. The gospel `gb_basis` therefore cannot link. **Fix:** remove the fiction; Module 76 deserializes the `*u8` system into Module 18 slots (`gb_new_poly`/`gb_append_term`), drives `gb_buchberger`, and serializes back via new M18 read accessors. (Same defect class as briefing §3.5's `ws_emit_fragment`/`cap_verify`/`at_now` fictions.)
2. **W2 — gospel `gb_basis` has 7 params, `gb_verify` 5.** **Fix:** fold `arena`/`p`/`order` into a module-scope session via `gbs_session(arena,p,order)`; fold `out_basis`/`out_cap`/`out_len` into one `out:*u8` aggregate (`[u32 cap][u32 len][bytes]`). Public sigs drop to ≤4 (mandate-driven; the only API divergence). Added `GBS_E_NOSESS` guard.
3. **Missing Module-18 read surface (serialization blocker).** Module 18 exposes only *leading*-term reads (`gb_lead_exp`/`gb_lead_coeff`); serializing a full basis polynomial needs full term iteration, which M18 keeps internal. **Fix:** require three trivial getters from Module 18 — `gb_poly_term_count(slot)`, `gb_term_at(slot,k,out_coeff,out_exps)`, `gb_coeff_lo(slot,k)` — that surface existing `GROEBNER_*` arrays. **Flagged NOT-YET-BUILT + Phase-2 addition to Module 18** (exact §3.5-defect-6 pattern). Wave: Module 18 must add these before Module 76 builds.
4. **M12/M17 — `gb_verify` is a no-op (`return GB_OK`).** The module's headline ("verified by independent checking") is unfulfilled; an arbitrary wrong basis passes. **Fix:** `gbs_verify` and the in-line `gbs_independent_verify` re-derive **ideal membership** — every original generator must `gb_reduce` to a zero-leading-coeff remainder modulo the basis — plus a monic spot-check and digest reproduction. KAT 3 proves the gate *rejects* a wrong basis (prove-the-negative). This is independent of the Buchberger path (M17 — never blindly trust synthesis).
5. **M19 — `budget` declared but ignored.** Gospel never reads `budget`; `GB_E_TIMEOUT` is dead. **Fix:** define a deterministic structural budget (`max_input_terms`/`max_basis_out`/`max_total_terms`, all `u64` counts — never wall-clock so the gate stays bit-identical) enforced at parse and post-closure; surface M18's `GROEBNER_E_FULL` as `GBS_E_FULL`. Replaces the meaningless timeout with a real, reproducible cost bound (M19).
6. **M6 — "verifier transcript enters the chain" never happens.** Gospel emits no witness. **Fix:** `gbs_emit_witness` routes through the real `wh_publish` (briefing §3.5 defect-2 — *not* the fictional `ws_emit_fragment`) carrying the reduced-basis content address (`gb_basis_digest`) as `out_commit`, the input commitment as `in_commit`, and the per-generator verdict transcript as payload; then `ws_register` anchors the fragment (Module 12). `gbs_witness_last` exposes the chain anchor (M6/M10).
7. **M2/W5 — no defined serialization ⇒ no bit-canonical output.** Gospel's `*u8` format is undefined. **Fix:** pin a magic-tagged little-endian byte format read/written byte-by-byte through `*u8` (Trap 5); Module 18 keeps terms in descending graded-lex so the stream is canonical for a given ideal+order (M2/W5). The output of two runs is byte-identical (KAT 2).
8. **M4 — silent monomial-order substitution risk.** Gospel passes `order:u8` straight into the fictional worker with no validation. **Fix:** `gbs_session` accepts only `GBS_ORDER_GRLEX (0)` and refuses (`GBS_E_FORMAT`) any other value rather than silently using graded-lex (M4 — exact, not "good enough").
9. **M5 — atomicity.** Define every failure path to set `out.len=0` and chain nothing; `GBS_OK ⟺ verified∧serialized∧chained`. No partial/garbage basis is ever delivered or witnessed (refusal, not bricking).
10. **NIH (M1) discipline confirmed.** Module 76 re-implements **no** field arithmetic, S-pair logic, reduction, or autoreduce — all delegated to Module 18 (and through it Module 04 galois / bigint). The only hand-rolled algorithm here is the FNV-1a input commitment (a reproducible content address, not a security primitive; soundness rests on the membership re-derivation, not on FNV collision-resistance). bigint construction (`bigint_from_u64`) is the sole direct field-layer touch, confined to the serialized boundary.
