## §1 — THE SUPERSESSION AND ITS LAW

WHAT RETIRES: external re-execution as the ground of trust. A script's epistemic form is
"re-run a foreseen check; trust = exit 0 today." That is slow (O(whole battery) per
change), external (the checker is bash, outside the organism), and closed-world (it can
only confirm what was foreseen when it was written) — fatal for a system whose purpose is
the **unforeseeable**: new ontologies, alien math, structures no test author could
anticipate.

WHAT IS CONSERVED, IN STRONGER FORM — every function the scripts serve is rehoused:

| script-era function          | organism-era carrier                                       |
|------------------------------|------------------------------------------------------------|
| regression detection         | closure-pin breakage (Ε3) + agon divergence (Ε2)           |
| determinism / fixpoint gate  | the covenant equation (Α3 — already scriptless)            |
| conformance history (KATs)   | sealed certificates, pinned in testament, O(ms) verify     |
| corpus meaning coverage      | the meaning ratchet (602, up-only) fed by the agon         |
| structural coverage audit    | `corpus_coverage` dark-surface, reference-source = agon    |
| build orchestration          | the metabolism organ over the sovbuild graph (Ε4)          |
| bootstrap from clean         | spore germination (Γ — already both ISAs)                  |
| falsifiability itself        | bearer independence + two-path proofs + live drains        |

**THE SUPERSESSION LAW (governs every deletion):** a script may be deleted only after
(1) its assertion is reachable as an organ verdict, and (2) the two-path proof has run —
break the invariant, watch the verdict REDDEN; restore it, watch it GREEN. No assertion
is dropped; every assertion is rehoused. A deletion without the two-path proof is a
tautology violation and the wave STOPS. Precedent already in the working tree:
`covenant_fold.sh` + `run_covenant_crosshost.sh` are **deleted today** because Α3 made
the equation the carrier. The pattern is proven once; Ε makes it total.

### §1.1 The truth criteria (why this is evidence, not faith)

1. **INDEPENDENCE.** Three separately-implemented bearers of `.iii` meaning — native
   `cg_r3` machine code, SVIR `eval`, route V (`svir_event` fold) — already agree on
   105/151 corpus classes. Agreement between independent bearers on a program *none has
   seen* is an oracle over the whole generated space; a KAT is a single point of it.
2. **TAMPER-EVIDENCE.** The testament (IIITSTMT v1, raw-byte tree over 3,649 files,
   SLH-DSA-signed, full-file deterministic) + `iii-witness` (distinct exit per tamper

3. **LIVE DRAINS.** Where a claim's space can't be sealed, the oracle is faced directly
   on live strides (Ω5: 2051/0/87 → 2071/0/67 measured in one run).
4. **TWO-PATH PROOFS.** Every organ verdict must be shown able to redden. A verdict that
   cannot fail is not evidence (the standing no-tautology law).
5. **ENVELOPE HONESTY.** Jurisdiction named, refusals counted by class (Ω4: 199,226
   decided / 8,130 capacity-refused why=1 / 4 unreached NAMED). No silent caps.

Old-math anchors (FIPS KATs, NIST vectors) are neither re-run nor deleted: their verdicts
were earned once, are sealed as certificates, pinned in the chain, verified in

every `sealed_channel` frame and every testament round-trips sign→verify in production
(`node_identity` `ni_*_sign`), so a break is a loud runtime event, never a silent drift.
