# THE WEAVE — Self-Model Consistency & the Dead-Import Gate

*The first concrete, gated increment of the approved plan's additive half ("close the self-management
loop"). Built from the advisor's redirect: don't run a vacuous objective over the call graph — diff III's
two independently-built self-models and surface where they disagree.*

## 1. The two self-models (the weave's substrate)

III holds **two** self-models, built by **independent** machinery:

- **Import graph** — `omnia/self_atlas` (generated by `STDLIB/scripts/gen_self_atlas.py`): the
  `extern … from "X.iii"` declaration edges between modules.
- **Call graph** — `sanctus/corpus_coverage` (its own code-level lexer): the actual caller→callee edges,
  built by walking the tree and lexing every `.iii` (nested comments / strings handled).

They are *supposed* to agree. Two independent organs agreeing (or not) is **a judge no one authored** —
the standard this project holds itself to.

## 2. The invariant

> **Every declared `extern … fn SYM … from "X.iii"` must name a symbol the module actually CALLS.**

A declared-but-never-called extern is **dead plumbing**: a phantom dependency edge in the self-model, and
the precise *"waste / NIH-breach that gets referenced"* that whole-module orphan scans structurally cannot
see (the module *is* used; the *import line* is dead). It is the sub-module-grain waste seven prior
detection axes missed.

## 3. The finding + the cleanup (commit `f2c9e8d5`)

Diffing the two graphs found **97 dead extern declarations across 58 files** (of 4027 intra-III externs).
Representative kinds:
- **Superseded**: `cad` declares the `sha256` *streaming* API (`init`/`update`/`final`) but calls only
  `sha256_oneshot`; `crypt_ed25519`'s `bigint_mod` was replaced by sovereign `ed_mod_l` ("was bigint_mod")
  with the extern left dangling.
- **Whole dead module-edges** (the module imports another and uses *nothing* from it): `uri→arena`,
  `ripple→kchain`, `crypt_ed25519→bigint_div`.
- **Vestigial accessors**: `dij_n`, `tc_var`, `pattern_id`, etc.

**Certified SAFE by construction**: an unused `extern` emits no reference, so removing it is
behavior- *and* link-neutral — the per-file recompile **is** the certificate. Verified: 58/58 touched files
recompile clean; 10/10 KATs green (`self_atlas` `1667` + 7 touched-module KATs + broad). The self-model was
regenerated to match the slimmed graph (660 nodes / 1323 edges; `cad` fan-in unchanged at 68). Reseal
deterministic (`SOURCES.mhash=ae5431e2`).

*Four measurement bugs were caught and fixed en route (commented-out extern; basename collision;
mixed path separators; CRLF line endings) — "suspect the measurement first." The underlying finding held.*

## 4. Enforcement — an III organ, NOT a Python script

A Python enforcer was prototyped and **deliberately removed**: III is sovereign and self-hosting, and a
Python gate living in the tree violates that ethos (no third-party-language enforcers). The honest,
sovereign enforcement path is **in III itself** — `sanctus/corpus_coverage` already lexes the whole tree and
builds the call graph in `.iii`; extending it (or a sibling `.iii` organ) to also flag a declared-but-never-
called extern is the in-language form of this invariant, and is the way it should be enforced. Until that
organ is built, the invariant is held by **manual review** of extern declarations against their call sites.
No Python lives in the tree for it.

## 5. FLAGGED — a latent bug surfaced, NOT auto-fixed (needs sign-off)

`aether/net.iii` calls `handle_close` (which *retains* the handle-table slot) but **never** `handle_drop`
(which *frees* it) — unlike sibling `aether/fs.iii`, which deliberately calls `handle_drop` to free slots
(its own comment documents the distinction). **`net` may leak handle-table slots over many connections.**
Wiring `handle_drop` into `net` is a *behavior change*, so it is **raised here for sign-off**, not applied —
the dead-import diff surfaced it, but the fix is a separate, deliberate decision.

## 6. Honest scope

This is **one** increment of the additive half: the self-model is now *consistent* (import graph == call
graph, gated) and *honest* (97 phantom edges removed). It is **not** the whole self-management loop — the
certified ripple actuators (`proof_ripple_unified`) running on the real graph remain future work, bounded by
the same discipline (a real, gated finding before any move; no vacuous objective; the green build certifies).
